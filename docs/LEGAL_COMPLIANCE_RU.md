## 1. Персональные данные (152-ФЗ)

### 1.1 Какие ПДн обрабатывает Near

| Категория | Примеры | Где в системе |
|-----------|---------|---------------|
| Учётная запись | email, хеш пароля | `users`, регистрация |
| Профиль | ФИО, должность, телефон, «о себе» | workspace `profile`, настройки |
| Компания | email, ФИО, должность сотрудников | `CompanyPage`, `/workspace/company` |
| Рабочие данные | задачи, комментарии, чат, тикеты | БД (tasks, comments, chat, support) |
| Технические | JWT, localStorage, офлайн-очередь, IP (на сервере) | frontend, логи сервера |
| Коммуникации | @mention, email-уведомления | `notifications`, SMTP |

### 1.2 Организационные меры (чеклист)

- [ ] Определить **оператора ПДн** (ИП / ООО)
- [ ] Подготовить **политику конфиденциальности** (обработка ПДн)
- [ ] Подготовить **согласие на обработку ПДн**
- [ ] Подготовить **пользовательское соглашение**
- [ ] Подать **уведомление в Роскомнадзор** (если нет оснований для исключения) — [pd.rkn.gov.ru](https://pd.rkn.gov.ru)
- [ ] Назначить **ответственного за организацию обработки ПДн** (приказ)
- [ ] Вести **реестр операций обработки ПДн**, локальные акты (по рекомендации юриста)
- [ ] Заключить **договоры поручения на обработку ПДн** с хостингом, SMTP, CDN, аналитикой (если появятся)

### 1.3 Локализация ПДн граждан РФ (242-ФЗ)

При сборе ПДн граждан РФ **первичная запись, хранение, уточнение** — в БД **на территории РФ**.

- [ ] Prod-БД и бэкапы разместить в **РФ** (Selectel, Yandex Cloud, VK Cloud, Timeweb и т.п.)
- [ ] Описать в политике **трансграничную передачу** (если подрядчики за рубежом)
- [ ] Согласовать с юристом схему при использовании зарубежного CDN только для статики

> **Риск:** деплой backend + SQLite/PostgreSQL на зарубежный VPS при аудитории в РФ — типичное несоответствие.

### 1.4 Технические меры в продукте (чеклист)

- [ ] Страница **`/legal/privacy`** (политика конфиденциальности) — ✅ шаблон в продукте
- [ ] Страница **`/legal/terms`** (пользовательское соглашение) — ✅ шаблон
- [ ] Страница или блок **`/legal/consent`** (текст согласия на обработку ПДн) — ✅ `/legal/consent`
- [ ] **Чекбокс при регистрации** (без предустановленной галочки) + ссылки на документы — ✅
- [ ] **Журнал согласий**: дата, версия политики, IP (хранить в БД) — ✅ `user_consent_logs`
- [ ] **Удаление аккаунта** в настройках + каскадное удаление/обезличивание данных — ✅
- [ ] **Экспорт данных пользователя** (JSON) — права субъекта ПДн — ✅ `GET /account/export`
- [ ] **Политика сроков хранения** (аккаунт, логи, архив задач, тикеты)
- [ ] HTTPS в production, секреты только через env (`JWT_SECRET` и др.)
- [ ] Минимизация полей (не собирать телефон/ФИО без необходимости)

## 5. Cookies, localStorage, Service Worker

Отдельного «cookie law» как в EU в РФ нет, но идентификаторы подпадают под 152-ФЗ.

- [ ] Описать в политике: JWT, `localStorage`, офлайн-очередь, Service Worker — цели и сроки
- [ ] При подключении **Яндекс.Метрики / аналитики** — упомянуть в политике и согласии
